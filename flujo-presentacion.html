SIG-I / Sistema Integral de Gestión de Incidencias
Flujo de la aplicación

Cómo se mueve el usuario, pantalla por pantalla

Recorrido funcional del sistema: desde el inicio de sesión hasta cada acción. Cada botón se traza hasta su acción de Redux, el endpoint de la API que consume y el resultado en pantalla.

02

Mapa de navegación

login → redirección por rol → paneles

ProtectedRoute vigila cada ruta: sin sesión → /login; con sesión pero rol no autorizado → /. El Foro es accesible por todos desde la Navbar.

03

Recorrido por pantalla

botón → acción Redux → endpoint → resultado
04

Ciclo completo de una incidencia

de la interfaz a la base de datos
Usuario → React/Redux
Inicia sesión. dispatch(login) guarda el JWT y redirige al panel según el rol.
React → API
POST /auth/login valida credenciales con bcrypt y responde JWT + usuario.
Usuario → React/Redux
Reporta una incidencia. dispatch(createTicket) → POST /tickets.
API → MongoDB
Crea el ticket asignando área y severidad automáticamente según el código INC.
API → tiempo real
Si la severidad es crítica, emite alerta por Socket.io al área y a Operaciones (≤10 s).
Admin de área → API
Avanza el estado del ticket. PATCH /tickets/:id/status exige comentario y queda en AuditLog.
